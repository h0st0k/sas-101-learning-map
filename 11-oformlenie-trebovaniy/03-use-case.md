# Use Case — сценарий использования

## Что такое Use Case
Детальное описание сценария взаимодействия пользователя с системой: шаг за шагом, включая альтернативные и ошибочные пути.

## Структура Use Case

| Элемент | Что писать |
|---------|------------|
| **Название** | Краткое описание сценария |
| **Актор** | Кто инициирует сценарий |
| **Предусловия** | Что должно быть выполнено ДО |
| **Основной поток** | Счастливый путь (минимум 6 шагов) |
| **Альтернативные потоки** | Другие пути к цели |
| **Исключительные потоки** | Ошибки — цель НЕ достигнута |
| **Постусловия** | Состояние системы ПОСЛЕ |

## Пример: Use Case «Оформление заказа»

**Название:** Оформление заказа
**Актор:** Клиент

**Предусловия:**
- Клиент авторизован
- Корзина не пуста

**Основной поток (9 шагов):**
1. Клиент нажимает «Оформить заказ» в корзине
2. Система показывает форму: адрес, телефон, способ оплаты, комментарий
3. Клиент заполняет адрес и телефон
4. Клиент выбирает способ оплаты: карта онлайн или наличные
5. Если наличные — клиент указывает сумму, с которой нужна сдача
6. Клиент нажимает «Заказать»
7. Система проверяет адрес — входит ли в зону доставки
8. Система создаёт заказ, присваивает номер, рассчитывает время доставки
9. Система показывает экран подтверждения: номер заказа, время доставки, статус «Принят»

**Альтернативный поток (шаг 4а):**
4а. Клиент выбирает «Наличные»
Система показывает поле «Приготовить сдачу с»
Клиент указывает сумму
→ Возврат к шагу 6

**Исключительные потоки:**
- Шаг 3а: Телефон не заполнен — система подсвечивает поле красным
- Шаг 3б: Телефон менее 11 цифр — «Введите номер полностью»
- Шаг 7а: Адрес вне зоны — «Доставка в этот район не осуществляется»
- Шаг 7б: Сервер не отвечает (>10 сек) — «Не удалось оформить. Попробуйте позже»
- Шаг 8а: Ошибка БД — «Заказ не создан. Попробуйте позже»

**Постусловия:**
- Заказ создан
- Клиент видит номер заказа и время доставки
- Заказ появился у администратора
