# SRS — спецификация требований к системе

## Что такое SRS
SRS (Software Requirements Specification) — единый документ со всеми требованиями к системе. «Единый источник правды» для всей команды.

## Структура SRS

### 1. Введение
- Цель системы
- Границы проекта
- Определения и сокращения

### 2. Общее описание
- Типы пользователей (роли)
- Основные функции
- Ограничения и допущения

### 3. Функциональные требования
- User Stories
- Use Cases
- Бизнес-правила
- Валидации

### 4. Нефункциональные требования
- Производительность (время отклика, нагрузка)
- Безопасность (шифрование, авторизация)
- Надёжность (время восстановления)
- Платформа (iOS, Android, веб)

### 5. Модели данных
- ER-диаграммы
- Словарь данных

### 6. API-контракты
- Swagger/OpenAPI спецификации

### 7. Требования к интерфейсу
- Макеты (Figma)
- Правила навигации

## Когда нужен SRS
- Крупный проект (Waterfall)
- Фиксированный бюджет и сроки
- Распределённая команда
- Регулируемая отрасль (финансы, медицина, госсектор)

## Когда НЕ нужен SRS
- Agile-команда с частыми релизами
- Небольшой проект
- Стартап с меняющимися требованиями

## Гибридный подход
User Stories в Jira + общий документ в Confluence с архитектурой и NFR.
