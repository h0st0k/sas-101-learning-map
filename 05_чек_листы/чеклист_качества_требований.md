# Чек-лист проверки требования

Прогоните требование через этот чек-лист перед передачей в разработку.

---

## ☑️ INVEST

- [ ] **Independent** — требование не зависит от других?
- [ ] **Negotiable** — обсуждаемое, не диктует реализацию?
- [ ] **Valuable** — несёт ценность для бизнеса/пользователя?
- [ ] **Estimable** — можно оценить трудозатраты?
- [ ] **Small** — умещается в одну итерацию?
- [ ] **Testable** — есть чёткие критерии приёмки?

---

## ☑️ Дополнительные критерии

- [ ] **Атомарность** — одно требование = одна мысль (нет «и»)?
- [ ] **Однозначность** — все понимают одинаково?
- [ ] **Полнота** — описаны альтернативные и исключительные потоки?
- [ ] **Трассировка** — есть источник и бизнес-цель?
- [ ] **Граничные случаи** — описано поведение при ошибках?

---

## ☑️ DoR (Definition of Ready)

- [ ] Ценность описана
- [ ] Acceptance Criteria готовы
- [ ] Нет блокирующих зависимостей
- [ ] Требование атомарное
- [ ] Дизайн/макеты готовы
- [ ] Команда оценила задачу
