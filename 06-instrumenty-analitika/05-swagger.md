# Swagger UI — документация API

## Для чего
- Описание REST-эндпоинтов
- Автогенерация документации
- Тестирование запросов

## Методы API
| Метод | Назначение |
|-------|------------|
| GET | Получить данные |
| POST | Создать данные |
| PUT | Обновить данные полностью |
| PATCH | Обновить часть данных |
| DELETE | Удалить данные |

## Что содержит спецификация
- URL эндпоинта (например, /api/users)
- Метод (GET, POST, PUT, DELETE)
- Параметры запроса (query, path, body)
- Коды ответов (200 — успех, 404 — не найдено, 500 — ошибка)
- Схема данных (JSON)

## Пример: GET /api/cats
- Параметры: type (square, small, xs)
- Ответ: картинка кота
