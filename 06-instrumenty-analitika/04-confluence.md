# Confluence — база знаний команды

## Для чего
- Хранение документации
- Протоколы встреч
- Требования (SRS, Use Cases)
- Архитектурные решения
- Глоссарии

## Структура
- **Разделы** (пространства) — для проектов или тем
- **Страницы** — документы
- **Заголовки** — для навигации (автооглавление)

## Особенности
- История версий (кто, когда, что менял)
- Экспорт в Word/PDF
- Интеграция с Jira (ссылки, макросы)
- Комментарии и обсуждения прямо на странице

## Пример страницы «Сервис профиль»
1. Описание сервиса
2. Архитектура (схема)
3. API-контракты
4. Use Cases
5. Интеграции
