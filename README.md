## 📚 Глоссарий (основные термины)

| Термин | Определение |
|--------|-------------|
| **Основной сценарий** | «Счастливый путь» — всё идёт по плану |
| **Альтернативный сценарий** | Другой путь к той же цели |
| **Исключительный сценарий** | Ошибка, цель НЕ достигнута |
| **Триггер** | Событие, запускающее сценарий |
| **Предусловие** | Что должно быть выполнено ДО начала |
| **Постусловие** | Состояние системы ПОСЛЕ завершения |
| **Атомарность** | Один шаг = одно действие |
| **User Story** | «Как [роль], я хочу [действие], чтобы [ценность]» |
| **Acceptance Criteria** | Проверяемые условия готовности |
| **DoR** | Definition of Ready — критерии готовности задачи |
| **SRS** | Software Requirements Specification — единый документ требований |
| **BPMN** | Нотация для моделирования бизнес-процессов |
| **UML** | Унифицированный язык моделирования |

## 🔄 Жизненный цикл системы (SDLC) — роль аналитика

| Этап | Задачи аналитика | Артефакты |
|------|------------------|-----------|
| **Инициация** | Оценка осуществимости, границы системы | Контекстная диаграмма |
| **Сбор требований** | Интервью, User Stories, Use Cases | Реестр требований, SRS |
| **Проектирование** | Модели данных, API-контракты | ER-диаграммы, Swagger |
| **Разработка** | Консультирование, управление изменениями | Актуализированная документация |
| **Тестирование** | Проверка Acceptance Criteria, негативные сценарии | Тест-кейсы |
| **Сопровождение** | Анализ инцидентов, доработки | Инструкции, новые требования |

## 🚀 Как использовать этот репозиторий

### 1. **Быстрый старт**
```bash
git clone https://github.com/ваш-username/sas-101-learning-map.git
cd sas-101-learning-map
2. Навигация
Откройте README.md — это главное меню

Переходите в нужную папку по ссылкам выше

3. Работа с шаблонами
Скопируйте шаблон из 03_шаблоны/

Вставьте в свой документ (Notion, Confluence, Word)

Заполните поля под свой проект

4. Подготовка к интервью
Откройте 05_чек_листы/чеклист_интервью.md

Спланируйте вопросы по кейсам из 04_кейсы/

5. Проверка требований
Возьмите любое требование

Прогоните через чек-лист из 05_чек_листы/чеклист_качества_требований.md
