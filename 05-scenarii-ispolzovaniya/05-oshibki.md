# Типичные ошибки при описании сценариев

| Ошибка | Чем опасна | Как исправить |
|--------|------------|---------------|
| Нет триггера | Непонятно, когда сценарий запускается | Указать явное событие |
| Пропущены предусловия | Разработчик не знает, с какого состояния начинать | Описать минимум 3 условия |
| Перепутаны потоки | Альтернативные и исключительные смешаны | Разделить: альтернатива → цель достигнута, исключение → нет |
| Неверный масштаб шагов | Слишком крупно («оформляет заказ») или слишком мелко | Одно действие на шаг |
| Нет возврата из ветки | После альтернативы непонятно, что дальше | Явно указать возврат |
| Не описаны взаимосвязи | Сценарии дублируются или противоречат | Использовать трассировку |

## Главное правило
На каждый шаг основного сценария задаём два вопроса:
1. «Как ещё можно?» → альтернативный сценарий
2. «Что может пойти не так?» → исключительный сценарий
