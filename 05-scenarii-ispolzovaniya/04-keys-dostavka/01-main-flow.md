# Кейс «Доставка продуктов» — Основной сценарий

## Контекст
Сервис доставки продуктов «Свежий». Клиент авторизован, корзина не пуста, все товары в наличии, оплата проходит, слот доставки доступен.

## Основной поток (10 шагов)

| Шаг | Действие |
|-----|----------|
| 1 | Клиент нажимает «Оформить заказ» в корзине |
| 2 | Система показывает форму: адрес, телефон, способ оплаты, интервал доставки |
| 3 | Клиент заполняет адрес и телефон |
| 4 | Клиент выбирает способ оплаты: «Карта онлайн» |
| 5 | Клиент выбирает интервал доставки (завтра, 12:00–14:00) |
| 6 | Клиент нажимает «Оплатить» |
| 7 | Система перенаправляет на платёжный шлюз — клиент вводит данные карты |
| 8 | Платёж успешен — система создаёт заказ и присваивает номер |
| 9 | Система показывает экран подтверждения (номер, состав, время, статус «Принят») |
| 10 | Система отправляет push-уведомление «Заказ №123 принят» |

## Предусловия
- Клиент авторизован
- Корзина не пуста
- Все товары есть на складе
- Платёжный шлюз доступен

## Постусловия
- Заказ создан в системе с уникальным номером
- Клиент видит экран подтверждения
- Клиент получил push-уведомление
- Заказ появился у сборщика
- Деньги списаны или заказ помечен «Оплата при получении»
