# REST API — основы

## Что такое API
API (Application Programming Interface) — интерфейс для взаимодействия между программами. Официант в ресторане: вы говорите, что хотите, он передаёт на кухню, приносит результат.

## REST — архитектурный стиль
REST (Representational State Transfer) — набор правил для построения API.

## Основные принципы REST
1. **Ресурсы** — всё, что можно получить или изменить (пользователи, заказы, товары)
2. **HTTP-методы** — действия над ресурсами
3. **Статус-коды** — результат операции (200 — успех, 404 — не найдено)

## HTTP-методы

| Метод | Назначение | Пример |
|-------|------------|--------|
| **GET** | Получить данные | GET /api/users — список пользователей |
| **POST** | Создать данные | POST /api/users — создать пользователя |
| **PUT** | Обновить полностью | PUT /api/users/1 — обновить пользователя |
| **PATCH** | Обновить частично | PATCH /api/users/1 — изменить только email |
| **DELETE** | Удалить данные | DELETE /api/users/1 — удалить пользователя |

## Статус-коды

| Код | Значение |
|-----|----------|
| 200 | OK — успешно |
| 201 | Created — создано |
| 400 | Bad Request — ошибка в запросе |
| 401 | Unauthorized — требуется авторизация |
| 403 | Forbidden — нет прав |
| 404 | Not Found — ресурс не найден |
| 500 | Internal Server Error — ошибка на сервере |
