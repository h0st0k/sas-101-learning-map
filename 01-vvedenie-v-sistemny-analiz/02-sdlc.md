# Жизненный цикл системы (SDLC)

## 6 этапов и роль аналитика

| Этап | Что делает аналитик |
|------|---------------------|
| **1. Инициация** | Анализ осуществимости, оценка трудозатрат, контекстная диаграмма |
| **2. Сбор требований** | Use Cases, User Stories, модели данных, API-контракты, NFR, граничные случаи |
| **3. Проектирование** | Маппинг данных, интеграционные потоки |
| **4. Разработка** | Отвечает на вопросы, управляет изменениями, обновляет документацию |
| **5. Тестирование** | Сверяет тест-кейсы с критериями приемки |
| **6. Внедрение и сопровождение** | Пишет инструкции, анализирует инциденты, собирает новые требования |
