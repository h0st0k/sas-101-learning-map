# ER-диаграммы — модели данных

## Что такое ER-диаграмма
ER (Entity-Relationship) — диаграмма, описывающая сущности (таблицы), их атрибуты (поля) и связи между ними.

## Сущности
- **Сущность** — объект, о котором храним данные (Книга, Читатель, Бронь)
- **Атрибут** — свойство сущности (Название, Автор, Статус)
- **Связь** — отношение между сущностями (Читатель → Бронь → Книга)

## Типы связей

| Тип | Обозначение | Пример |
|-----|-------------|--------|
| **Один-к-одному** | 1:1 | Человек ↔ Паспорт |
| **Один-ко-многим** | 1:N | Читатель → Бронь (один читатель может иметь много броней) |
| **Многие-ко-многим** | N:N | Книга ↔ Автор (у книги много авторов, автор пишет много книг) |

## Пример ER-диаграммы: Библиотека

┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│ Читатель │ │ Бронь │ │ Книга │
├─────────────┤ ├─────────────┤ ├─────────────┤
│ ID (PK) │──1───N│ ID (PK) │──N───1│ ID (PK) │
│ Имя │ │ Читатель_ID (FK)│ │ Название │
│ Телефон │ │ Книга_ID (FK) │ │ Автор │
│ Email │ │ Дата_брони │ │ Статус │
└─────────────┘ │ Статус │ │ Год │
└─────────────┘ └─────────────┘

**Расшифровка:**
- **PK** — Primary Key (уникальный идентификатор)
- **FK** — Foreign Key (ссылка на другую таблицу)

## Связи в примере
1. **Читатель (1) → Бронь (N)** — один читатель может иметь много броней
2. **Книга (1) → Бронь (N)** — одна книга может быть забронирована много раз
