# Требования к системе — «Прокачу»

## Функциональные требования (FR) — 15 шт.

| ID | Требование |
|----|------------|
| FR-01 | Система отображает карту с пинами 20 станций и статусами велосипедов (свободен / занят) |
| FR-02 | Система позволяет забронировать велосипед на 15 минут (с таймером и push за 3 минуты) |
| FR-03 | Система проверяет, что велосипед свободен перед бронированием |
| FR-04 | Система отображает стоимость аренды (почасовой / суточный) |
| FR-05 | Система позволяет оплатить аренду (карта / СБП) через PCI DSS-шлюз |
| FR-06 | После оплаты система разблокирует замок (BLE) и присваивает статус «Активен» |
| FR-07 | Система отслеживает время аренды и геолокацию велосипеда |
| FR-08 | При остановке на станции (>60 сек) система автоматически завершает аренду |
| FR-09 | При завершении аренды система меняет статус на «Завершён» и отправляет чек |
| FR-10 | Система сохраняет историю аренд клиента (12 месяцев) |
| FR-11 | Система позволяет клиенту повторить аренду из истории |
| FR-12 | Система позволяет отправить отчёт о поломке (выпадающий список компонентов) |
| FR-13 | Система блокирует аренду после 23:00 (сообщение: «Прокат закрыт до 06:00») |
| FR-14 | При выезде за МКАД система отправляет push-требование вернуться в зону |
| FR-15 | Система логирует все действия (бронирование, оплата, завершение) |

## Нефункциональные требования (NFR) — 6 шт.

| ID | Требование |
|----|------------|
| NFR-01 | Приложение доступно на iOS (15.0+) и Android (10.0+) |
| NFR-02 | Время обработки запроса — не более 1.2 секунды (95-й перцентиль) |
| NFR-03 | Нагрузка — до 300 активных сессий в пиковые часы |
| NFR-04 | Интеграция с платёжным шлюзом по стандарту PCI DSS |
| NFR-05 | Бюджет — не более 2 000 000 руб. |
| NFR-06 | Срок реализации — 3 месяца (90 дней) |
