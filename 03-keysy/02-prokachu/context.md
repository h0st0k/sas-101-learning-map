# Кейс «Прокачу» — аренда велосипедов

## Контекст
- Сервис аренды велосипедов
- Сейчас: Telegram-бот (8 шагов, ручное завершение)
- Хотят: мобильное приложение (iOS + Android)
- 200 велосипедов, 20 станций
- Бюджет: 2 млн руб., срок: 3 месяца

## Проблемы текущего бота
1. 8 шагов — пользователи теряются
2. Нет интерактивной карты — только список
3. Ручное завершение — забывают, поддержка возвращает деньги
4. Нет бронирования
5. Нет истории поездок
6. Нет отчёта о поломках

## Роли
1. Клиент — ищет, бронирует, арендует
2. Оператор поддержки — возвращает деньги, обрабатывает обращения
3. СТО — отвечает за архитектуру, масштабирование
