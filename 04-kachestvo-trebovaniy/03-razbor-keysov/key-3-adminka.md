# Кейс 3 — Требование про админку

## Исходное требование
«Администратор должен иметь возможность управлять пользователями: добавлять, удалять, блокировать, редактировать роли и права доступа. Интерфейс должен быть интуитивно понятным и отзывчивым. Система должна обеспечивать высокий уровень безопасности при работе с пользовательскими данными»

## Нарушения
| Проблема | Критерий |
|----------|----------|
| 5 действий в одном требовании | Атомарность + Small |
| «Интуитивно понятный» | Testable + Однозначность |
| «Отзывчивый» | Testable |
| «Высокий уровень безопасности» | Однозначность + Testable |
| Нет граничных случаев | Полнота (админ удаляет себя) |
| Нет трассировки | Valuable (откуда требование про безопасность?) |

## Исправленный вариант
**5 атомарных требований:**
1. Администратор добавляет пользователя
2. Администратор удаляет пользователя
3. Администратор блокирует пользователя
4. Администратор редактирует роли
5. Администратор редактирует права доступа

**Граничные случаи:**
- Админ не может удалить свою учётную запись
- Нельзя заблокировать последнего администратора
- Все действия записываются в лог

**Нефункциональные:**
- Время отклика — не более 1 секунды
- Данные зашифрованы
- Вход через двухфакторную аутентификацию

**Трассировка:** требования безопасности на основе 152-ФЗ
